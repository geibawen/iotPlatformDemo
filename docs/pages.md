# 页面功能说明

## 1. 概览页（Dashboard）

**路径**：`/`

展示平台整体概况：
- **统计卡片**：产品数量、插件数量、推送规则数、推送日志数
- **产品列表**：最近创建的产品快速预览
- **最近日志**：最新推送日志记录

---

## 1.1 功能拓扑（Roadmap）

**路径**：`/roadmap`

- 左侧菜单新增 `Roadmap` 入口
- 页面展示网站整体功能拓扑图（SVG）
- 图中覆盖三类角色视角（开发者 / 测试人员 / 系统管理员）
- 展示前端能力层、后端路由层、共享类型层、JSON 存储层的关系

---

## 2. 产品管理

### 2.1 产品列表（ProductList）

**路径**：`/products`

- 表格展示所有产品，支持按名称搜索
- 操作：创建、编辑、删除、查看详情
- 产品表单（Modal）：名称、分类、连接方式、图片 URL、描述
- 创建时支持“基于产品创建（可选）”：可继承源产品的整套物模型（服务、属性、动作）
- 选择继承来源后，表单会实时显示将继承的服务/属性/动作数量预览

### 2.2 产品详情（ProductDetail）

**路径**：`/products/:id`

顶部展示产品基本信息和状态，下方 Tab 页切换：

| Tab | 内容 |
|-----|------|
| 基本信息 | 产品字段的只读展示 |
| 属性（Property） | 查看/编辑/删除全部属性（新增需在服务内进行） |
| 动作（Action） | 查看/编辑/删除全部动作（新增需在服务内进行） |
| 服务（Service） | 服务 CRUD + 服务内新增/删除属性与动作 |
| 关联插件 | 展示关联到该产品的所有插件 |

---

## 3. 物模型编辑

物模型编辑器嵌入在产品详情页的 Tab 中，分为三部分：

### 3.1 属性编辑（PropertyTable + PropertyForm）

属性表格列：标识符、名称、数据类型、读写模式、单位、是否必填

说明：
- 本 Tab 保留“查看全部属性”能力
- 不提供直接新增入口（需先创建服务）
- 新增属性在“服务 (Service)”Tab 的具体服务下完成

**属性表单**是最复杂的表单，根据选择的数据类型动态渲染不同的 specs 字段：

| 数据类型 | 动态字段 |
|---------|---------|
| int / float | 最小值、最大值、步长、单位 |
| bool | 真值标签、假值标签 |
| string | 最大长度 |
| enum | 枚举值列表（动态增减行） |
| struct | 结构体字段列表（动态增减行） |
| array | 数组元素类型 |

### 3.2 动作编辑（ActionTable + ActionForm）

动作表格列：标识符、名称、输入参数数、输出参数数

说明：
- 本 Tab 保留“查看全部动作”能力
- 不提供直接新增入口（需先创建服务）
- 新增动作在“服务 (Service)”Tab 的具体服务下完成

表单包含：
- 基础信息：标识符、名称、描述
- 输入参数列表（Form.List 动态增减）
- 输出参数列表（Form.List 动态增减）

### 3.3 服务编辑（ServiceTable + ServiceForm）

服务表格列：标识符、名称、关联属性数、关联动作数

服务表单包含：
- 基础信息：标识符、名称、描述

服务内容管理包含：
- 在某个服务下新增属性
- 在某个服务下新增动作
- 显示该服务下所有属性与动作（Tag 列表）
- 在服务内删除某个属性或动作

服务详情弹窗：
- 服务列表提供“详情”按钮
- 点击后弹窗展示当前服务详情
- 弹窗内分为“属性”“动作”两个区域（结构类似对应 Tab）
- 支持在弹窗内新增、编辑、删除该服务下的属性和动作

## 4. 插件管理

### 4.1 插件管理入口（侧边栏二级菜单）

**主路径**：`/plugins`（自动重定向到 `/plugins/device`）

二级菜单（左侧导航）：

| 标签 | 功能 | 说明 |
|------|------|------|
| 设备插件管理 | 设备插件 CRUD | 支持 iOS、Android、鸿蒙多平台 |
| 功能插件管理 | —（占位） | 敬请期待 |

### 4.2 设备插件列表（PluginList）

**路径**：`/plugins/device`

- 表格展示：插件名称、类型、平台支持（多个 Tag）、关联产品、状态、更新时间
- 操作：创建、编辑、删除、查看详情
- 顶部新增“插件拓扑图”按钮：可视化展示所有设备插件与产品的对应关系，并直观看到插件是否上线

### 4.3 功能插件列表（FunctionalPluginList）

**路径**：`/plugins/functional`

- 占位页（敬请期待）
- 提供禁用态“新建功能插件”按钮

**插件表单**（PluginForm）：
- **插件名称**（必填）
- **插件描述**
- **插件类型**（必填）：功能插件、设备插件（单选下拉）
- **支持平台**（必填）：
  - iOS（默认选中，禁用）
  - Android（默认选中，禁用）
  - HarmonyOS（可选，默认未选）
  - 验证：必须至少选中 iOS 和 Android
- **关联产品**（穿梭框 Transfer）

### 4.4 插件详情（PluginDetail）

**路径**：`/plugins/:id`

顶部展示插件信息和启用/停用开关，下方 Tab：

| Tab | 内容 |
|-----|------|
| 基本信息 | 插件字段的只读展示（类型、平台支持） |
| 版本管理 | 版本列表和发版操作 |

### 4.5 版本管理（VersionManager）

**版本表格列**：版本号、平台文件（按平台分组显示）、状态、发布说明、创建时间

**功能**：
- **新建版本**：
  - 版本号（语义化版本格式，必填）
  - 发布说明（可选）
  - **为每个平台上传文件**（根据插件选定的平台动态显示上传点，每个平台一个字段）
    - 例如：插件配置为 iOS + Android，则显示两个文件上传字段
    - 所有平台的文件都必须上传，否则提交失败
    - 文件格式可为 `.ipa`（iOS）、`.apk`（Android）、`.hap`（鸿蒙）
  - 最大单文件 50MB

- **版本状态流转**：
  - draft（草稿）→ waiting_test（待测试）
  - 批准后 approved（测试通过）→ online（已上线）
  - online ↔ offline（已下线）
  - 仅 draft 和 rejected 状态可删除

## 5. 推送配置

### 5.1 推送规则（PushRules）

**路径**：`/push/rules`

- 表格展示规则列表，支持 Switch 快速开关
- 规则表单：
  - 规则名称
  - 关联产品（下拉选择）
  - 触发类型（设备上线/离线/属性变更/事件上报）
  - 触发条件（动态条件行：字段 + 操作符 + 值）
  - 推送通道（下拉选择已创建的通道）
  - 消息模板（下拉选择已创建的模板）

### 5.2 推送通道（PushChannels）

**路径**：`/push/channels`

- 表格展示通道名称、类型（Tag 颜色区分）、状态、操作
- **连接测试**：点击"测试"按钮模拟连接测试（2 秒延迟 + 成功提示）
- 通道表单根据类型动态渲染不同配置字段：
  - **HTTP 回调**：URL、Method、Headers、Secret
  - **MQTT**：Broker 地址、端口、Topic、用户名、密码
  - **Kafka**：Brokers、Topic、Group ID
  - **MNS**：Endpoint、AccessKeyId、AccessKeySecret、Topic

### 5.3 消息模板（MessageTemplates）

**路径**：`/push/templates`

- 表格展示模板列表
- 模板表单：名称、描述、格式（JSON/TEXT）、模板内容
- **变量插入**：点击变量按钮自动插入占位符到内容编辑器
- **模板预览**：查看模板内容（代码高亮展示）

### 5.4 推送日志（PushLogs）

**路径**：`/push/logs`

- 表格展示推送记录，支持按状态筛选（全部/成功/失败）
- **详情抽屉（Drawer）**：查看完整的请求体、响应体、错误信息
- 操作：刷新、清空日志

## 6. 角色模拟与切换

### 6.1 右上角角色切换

各角色布局右上角头像下拉菜单支持角色切换：

- 切换到开发者：`/dashboard`
- 切换到测试人员：`/tester/plugins`
- 切换到系统管理员：`/admin/product-categories`

### 6.2 测试人员工作台（TesterLayout）

**路径**：`/tester`

- 插件测试：`/tester/plugins`
- 白名单管理：`/tester/whitelist`

### 6.3 系统管理员工作台（AdminLayout）

**路径**：`/admin`

- 产品分类管理：`/admin/product-categories`
- 支持产品分类增删改查
